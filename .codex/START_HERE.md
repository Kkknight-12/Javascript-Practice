# Codex Start Here

Use this folder as the working memory for the JavaScript practice repo.

## Current Direction

This repo is a public JavaScript learning/practice collection with two separate
areas:

```text
src/       -> terminal-first runnable examples and nearby notes
projects/  -> compact vanilla TypeScript projects with JavaScript-first reasoning
```

Do not turn the note collection into a website. Browser project work belongs in
its own folder under `projects/`.

## Read Order

1. `PRACTICAL_PROJECT_MEMORY.md` when working in project-based practice mode
2. `PROJECT_MEMORY.md` before creating or revising teaching notes
3. `STRUCTURE_REVIEW.md`
4. `TASK_TRACKER.md`

## Working Modes

- Project mode uses compact feature tickets to practise the complete JavaScript
  topic inventory in realistic situations.
- Note mode preserves the existing learner-facing `.js` plus `.md` workflow.
- When a project exposes a missing or weak note, repair the note first and then
  resume the same project ticket.

## Current Story

The current note sprint remains reviewable independently from the planned first

## Working Rules

- Work in small sprints.
- Keep a visible checklist before coding.
- Stop after each sprint for review.
- Do not commit, push, merge, or delete tracked content without confirmation.
- Avoid mass file moves while the worktree has unrelated uncommitted changes.
- Keep browser tooling inside the selected project folder.
- Do not add a framework or transform the `src/` note collection into a site.
